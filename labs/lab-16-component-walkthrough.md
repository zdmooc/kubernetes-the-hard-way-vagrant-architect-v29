# Lab 16 — Walkthrough composant par composant

## Objectif
Faire un tour complet du cluster en lisant Kubernetes comme un architecte.

## Parcours
1. API Server
2. etcd
3. scheduler
4. controller-manager
5. kubelet
6. kube-proxy
7. CoreDNS
8. CNI
9. stockage
10. RBAC

## Livrable
Pour chaque composant, remplir :
- rôle,
- flux,
- preuve de fonctionnement,
- risque principal,
- point d’audit.
