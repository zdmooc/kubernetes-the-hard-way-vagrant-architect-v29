# Release Notes V30

La V30 introduit le parcours Ansible en parallèle du parcours shell historique.

## Nouveautés

- **Parcours Ansible** : Ajout d'une structure complète de rôles, inventaires et playbooks pour automatiser le déploiement de Kubernetes.
- **Documentation** : Comparaison des approches shell vs Ansible, et modèle opérationnel Ansible.
- **Lab** : Nouveau lab 29 pour pratiquer le déploiement Ansible.

## Ce qui est préservé

- **Parcours Shell** : Intact et fonctionnel.
- **Topologie Vagrant** : Aucune modification de la topologie existante.

## Objectifs atteints

- Fournir une alternative industrielle pour la construction et la gestion de la plateforme.
- Préparer le terrain pour les futures couches (AWX, CI/CD).
