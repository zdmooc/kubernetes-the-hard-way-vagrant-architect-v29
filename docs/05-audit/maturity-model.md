# Maturity model

## Niveau 1 — Fonctionne
Le cluster répond aux commandes de base.

## Niveau 2 — Exploitable
Runbooks, sauvegardes, smoke tests et supervision minimum.

## Niveau 3 — Gouverné
RBAC, quotas, policies, séparation des responsabilités.

## Niveau 4 — Résilient
HA, PDB, anti-affinity, stratégie DR documentée.

## Niveau 5 — Industrialisé
GitOps, SLOs, conformité, audit continu, architecture standardisée.
