# Contributing

## Règles
- garder une orientation architecture + exécution + audit ;
- expliquer chaque ajout avant d'automatiser ;
- documenter les hypothèses et limites ;
- ajouter une validation et un angle diagnostic.


## Naming conventions

- ADRs must follow a unique incremental sequence.
- Labs must follow a unique incremental sequence.
- Release notes must live under `docs/10-release/`.
